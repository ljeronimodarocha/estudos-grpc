## AGENTS.md

### Commands
- `mvn install` for building dependencies
- `mvn test` to run all tests
- `mvn clean` to clean build artifacts
- `mvn clean test` to clean and run tests
- `mvn package` for packaging application

### Test Execution
- Run tests: `mvn test`
- Coverage: `mvn jacoco:report` (60% minimum coverage)
- Focus on module: `mvn test -pl user` (replace 'user' with module name)
- Auth tests: `mvn test -pl auth`
- Book tests: `mvn test -pl book`
- User tests: `mvn test -pl user`

### Monorepo Structure
- `User/`: Spring Boot application module with all about users
- `Book/`: Spring Boot application module with all about Books
- `Auth/`: Spring Boot application module with security and authentication module
- `contratos-grpc/`: gRPC contract definitions and code generation

### Framework Quirks
- Maven-based build system
- Shared parent pom: `sistema-grpc-parent` (version 1.0.0)
- Java version: 21
- Spring Boot version: 4.1.0
- gRPC integration: All modules use `spring-boot-starter-grpc-server` and `spring-boot-starter-grpc-client`
- Proto definitions centralized in `contratos-grpc/src/main/proto/`
- Protobuf-Maven-Plugin for gRPC code generation
- Jacoco configured for code coverage in all modules

### Security Architecture
- **Auth**: Central authentication service exposing gRPC endpoints for token validation
  - `/auth/validate/**` - Public endpoint for JWT validation via gRPC
  - `/auth/login`, `/auth/register`, `/auth/refresh`, `/auth/logout` - Public endpoints
  - Uses JwtUtil for token validation
- **Book/User**: Delegate token validation to Auth via gRPC
  - `JwtValidationFilter` validates tokens by calling Auth service
  - `GrpcClientFactory` creates gRPC stubs to connect to Auth (localhost:9090)
  - All endpoints require JWT authentication
- **Architecture**: Delegated authentication pattern where service modules (Book, User) delegate token validation to Auth service via gRPC

### Testing Conventions
- Unit tests: Maven Surefire plugin
- Jacoco: Code coverage reports available with `mvn jacoco:report` (60% minimum coverage)
- gRPC tests: Proto files generated by `mvn generate-sources` in contratos-grpc module
- Integration tests: May require running `mvn test` with specific profiles
- gRPC code generation: Copy generated files from `contratos-grpc/target/generated-sources/protobuf/com/example/grpc/auth/` to module source directories
- Test structure:
  - Auth module: JwtUtilTest, UserRepositoryAuthenticationTest, TokenRepositoryTest
  - Book module: BookServiceImplTest
  - User module: UserServiceTest
